io.js와 Node.js 소식

이번 주에는 io.js 릴리스가 없었습니다. 전체 변경사항은 GitHub에서 확인할 수 있습니다.

커뮤니티 업데이트

  • ES2015가 승인되었습니다. ES2015는 이제 ECMA 표준입니다.
  • Linux 재단이 Node.js Foundation을 발표했습니다.
  • IBM의 부사장인 Angel Diaz가 새 Node.js 재단에 대해 이야기를 했습니다. YouTube에서 확인하세요.
  • Raymond Camden의 블로그 포스트: 클라우드 코드를 사용하기, 모바일 애플리케이션 보안, Node.js와 Bluemix
  • Strongloop의 글: strong-pm.io를 이용한 여러 개의 Node.js 앱 배치하기
  • Strongloop의 글: Node + React로 Isomorphic JavaScript 구현하기, 왜 이게 뜨는가?
  • O’Reilly Radar, Leah Hunter의 글: 스마트 기기를 가장 똑똑하게 프로그램하는 방법: Node.js
  • Sebastian Seilund의 글: 비동기 Node.js 실습하기: 무엇을 언제 사용해야 할까?
  • DailyJS의 블로그 포스트: 진정한 단일 페이지 앱이 아니다.
  • Strongloop의 글: Docker, StrongLoop로 Node.js 앱 컨테이너 만들기
  • Node.js가 사용된 곳들: Node를 사용한 스타트업, 앱, 프로젝트들
  • Free Software Magazine: 재단과 합병에 관한 인터뷰
  • Daniel Khan의 글: 왜 Node.js가 기업 시장에서 널리 알려지고 성공하게 되었는가, Node.js를 이용해 간단하게 유지하기
  • Paul Sauve의 블로그 포스트: NodeJS 통합 개발 환경(IDEs) 비교 (ST3와 WebStorm!)
  • Nodesource의 글: npm 이해하기와 그에 대한 블로그 포스트
  • Box의 글: Node.js의 높은 가능성
  • Nodevember의 글: Docker를 이용한 확장 가능한 Node.js 애플리케이션 배치하기. YouTube에서 보실 수 있습니다.
  • Modulus.io의 글: Node.js를 사용하는 이유 톱 10
  • Leankit의 글: Painless Microsoft SQL Server with Node.js and seriate
  • 블로그 포스트: 당신의 기업환경 프로젝트를 소개하기 위한 황금률
  • Rod Vagg의 글: 기업 생태계 안에서 Node의 상태
