Node v9.0.0(현재 버전)

  • 작성자: James M Snell
  • 원문: Node v9.0.0 (Current)
  • 번역: BaroqueCoding

주요 변경사항

  • Async hooks

    • 오래된 실험적 API들이 제거되었습니다. [d731369b1d] #14414

  • Errors

      • fs: [219932a9f7] #15043, [b61cab2234] #11317

  • Child Processes

    • 프로세스의 nextTick에서 오류 이벤트를 발생시킵니다. [f2b01cba7b] #4670

  • Domains

    • 오랫동안 폐기 예정이었던 .dispose() 메서드가 제거되었습니다. [602fd36d95] #15412

  • fs

    • fs.ReadStream과 fs.WriteStream 클래스들은 이제 destroy()를 사용합니다. [e5c290bed9] #15407
    • fs 모듈의 콜백은 이제 undefined 컨텍스트에서 호출됩니다. [2249234fee] #14645

  • HTTP/1

    • 파싱에 실패했을 때 400 Bad Request 응답이 발생합니다. [f2f391e575] #15324
    • 소켓 커넥션이 일어났을 때 타임아웃이 설정됩니다. [10be20a0e8] #8895
    • 'error' 이벤트를 두 번 발생시키는 Request 버그가 수정되었습니다. [620ba41694] #14659
    • HTTP 클라이언트는 net.Socket뿐만 아니라 범용 Duplex 스트림도 사용할 수 있습니다. [3e25e4d00f] #16267

  • Intl

    • 폐기예정이었던 Intl.v8BreakIterator가 제거되었습니다. [668ad44922] #15238

  • OS

    • os.EOL 속성은 이제 읽기 전용입니다. [f6caeb9526] #14622

  • Timers

    • setTimeout()이 받는 timeout 값이 부호 없는 32비트 정수범위를 초과하는 경우 경고가 발생합니다. [ce3586da31] #15627
